# SKU 利润计算器设计

日期：2026-06-20

## 1. 目标

在现有 `/sku-details` 页面内加入利润计算器。选中 SKU 后，计算器复用主数据中的采购价与商品毛重，并允许运营人员调整销售价格，实时查看预计利润、利润率、成本构成，以及 0%、20%、40% 目标利润率对应的销售价格。

## 2. 已确认交互

1. 采购价和商品毛重跟随当前选中 SKU，只读展示，不允许手工修改。
2. 销售价格默认使用 SKU 最低售价，允许手工修改。
3. 切换 SKU 时，销售价格重置为新 SKU 的最低售价。
4. 利润计算器位于“规格价格”之后、“库存销售”之前，保持现有 SKU 详情页的信息层级。
5. 汇率、物流、仓储和平台费率收在可展开的“成本参数”区域，默认沿用参考 Excel。
6. 普通用户只能查看成本参数；管理员可以编辑、预览、恢复默认并保存为全局参数。
7. 利润测算结果只在当前页面一次性展示，不保存、不写入飞书。
8. 管理员保存的成本参数不写入飞书；其他用户在下次进入或刷新页面时读取最新参数，不要求实时推送。

## 3. 默认参数

| 参数 | 默认值 |
| --- | ---: |
| USD/CNY 汇率 | 6.76 |
| 头程运费 | 20 RMB/kg |
| 尾程运费 | 5 USD/kg |
| 关税比例 | 27.5% |
| 海外仓入库费 | 1.50 USD |
| 海外仓操作费 | 3.00 USD |
| eBay 佣金比例 | 13.25% |
| 广告费比例 | 17% |
| 退货率比例 | 3% |

参考表中的四档头程、尾程单价当前完全相同，因此一期按单一重量单价建模；后续若费率分档出现差异，再扩展为阶梯规则。

## 4. 计算口径

设：

- `purchasePriceCny`：采购价（RMB）
- `weightKg`：商品毛重（g）/ 1000
- `salePriceUsd`：销售价格（USD）
- `variableRate`：eBay 佣金率 + 广告费率 + 退货率

各成本：

1. 产品成本（USD）=`purchasePriceCny / exchangeRate`
2. 头程运费（USD）=`weightKg * firstMileCnyPerKg / exchangeRate`
3. 尾程运费（USD）=`weightKg * lastMileUsdPerKg`
4. 关税金额（USD）=`产品成本（USD）* tariffRate`
5. 固定成本=`产品成本 + 头程 + 尾程 + 关税 + 入库费 + 操作费`
6. eBay 佣金=`salePriceUsd * ebayFeeRate`
7. 广告费=`salePriceUsd * adRate`
8. 退货成本=`salePriceUsd * returnRate`
9. 总成本=`固定成本 + eBay 佣金 + 广告费 + 退货成本`
10. 净利润=`salePriceUsd - 总成本`
11. 净利率=`净利润 / salePriceUsd`

参考表的关税公式错误引用了 eBay 佣金比例；本功能按已确认口径修正为“产品成本（USD）× 关税比例”。

目标利润率售价：

`targetPrice = 固定成本 / (1 - variableRate - targetMargin)`

当分母小于等于 0 时，该目标不可达，界面显示 `--`。

## 5. 页面结构

### 5.1 顶部输入与结果

- 只读：采购价（RMB）、毛重（g）
- 可编辑：销售价格（USD）
- 核心结果：预计净利润、预计净利率、总成本
- 根据利润正负使用绿色、橙色或红色状态色，但不改变页面整体的浅色运营后台风格。

### 5.2 目标售价

横向展示三项：保本价（0%）、20% 利润率售价、40% 利润率售价。桌面端三列，移动端保持可读的单列或三列自适应布局。

### 5.3 成本构成

按产品成本、头程、尾程、关税、入库、操作、eBay 佣金、广告、退货逐项显示：

- 金额（USD）
- 占当前销售价格的比例
- 轻量水平比例条

### 5.4 成本参数

默认折叠。普通用户展开后看到只读参数和“仅管理员可修改”提示，不显示保存或恢复默认命令。管理员展开后可以：

1. 编辑参数并立即预览当前 SKU 的利润结果。
2. 点击“恢复默认”重置当前草稿；恢复动作本身不影响其他用户。
3. 点击“保存并应用”把当前参数写入站点级配置；保存成功后其他用户在下次进入或刷新页面时读取。

销售价格、当前 SKU 和利润结果均不保存。

## 6. 边界与错误状态

1. 未选择 SKU 时不渲染计算器。
2. 采购价、毛重或售价缺失/小于等于 0 时，结果显示不可计算提示，不产生 `NaN` 或无穷值。
3. 百分比参数使用 0–100 的界面输入，在计算层转换为小数。
4. 汇率必须大于 0；无效输入回退为不可计算状态。
5. 所有金额展示保留两位小数，利润率展示一位小数。
6. 成本参数读取失败时回退到代码默认值，并显示“当前使用默认参数”的提示。
7. 管理员保存失败时保留当前草稿并显示错误，不覆盖已经保存的全局参数。
8. 客户端的管理员状态只控制界面；服务端保存接口必须再次调用 `requireAdmin()`，普通用户直接请求保存接口仍返回 403。

## 7. 技术方案

1. 新增 `src/lib/profit-calculator.ts`，集中维护默认参数、纯计算函数和结果类型。
2. 新增 `src/components/sku/profit-calculator.tsx`，负责 SKU 绑定、参数编辑和可视化展示。
3. 在 `src/app/(main)/sku-details/page.tsx` 中挂载组件，不改变现有飞书读取流程。
4. 新增 `src/lib/profit-settings.ts`，负责成本参数校验与持久化：本地开发使用 `data/profit-settings.json`，Netlify 运行时使用站点级 Blobs；不调用飞书 API。
5. 新增 `/api/profit-settings`：所有已登录用户可 `GET`，只有通过 `requireAdmin()` 的管理员可 `PUT`。
6. `data/profit-settings.json` 加入 Git 忽略和发布安全检查，避免本地运行数据进入版本库。
7. 使用 Vitest 覆盖参考 Excel 示例、关税修正、目标售价、无效输入、参数校验、普通用户保存被拒绝和管理员保存成功。

## 8. 数据流与权限

1. 页面进入时并行请求 `/api/profit-settings` 与 `/api/auth/me`。
2. 成本参数接口返回已保存参数；首次使用时返回代码默认值。
3. 普通用户用服务端参数计算当前页面结果，输入控件为只读。
4. 管理员编辑的是本地草稿；保存前只影响当前页面预览。
5. 管理员点击“保存并应用”后调用 `PUT /api/profit-settings`，服务端校验会话、管理员角色和全部字段范围，再原子写入全局配置。
6. 其他已打开页面不轮询、不实时推送；刷新或重新进入后重新读取。
7. 任何利润测算输入和结果都不通过该接口传输或持久化。

## 9. 验证

1. 先运行利润计算器单元测试，再运行全量测试和 lint。
2. 使用本地 `develop` 服务在内置浏览器验证选中 SKU、切换 SKU、修改售价、展开参数和恢复默认。
3. 验证桌面与移动视口，确保文字、输入、结果和比例条不重叠。
4. 不执行 Netlify 部署，不推送 GitHub。
5. 分别以管理员和普通用户验证成本参数的可编辑状态与保存权限，并直接请求接口验证服务端 403 防护。
