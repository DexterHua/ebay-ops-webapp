# 库存批次流转与飞书自动化设计

## 1. 背景

当前系统已经将 SKU 基础资料、库存策略和运营汇总拆开：

- `01_SKU主数据`：SKU 静态基础资料。
- `02_库存流水`：库存变动记录。
- `18_SKU库存策略`：安全库存、补货周期等策略参数。
- `19_SKU运营汇总`：本地库存、橙联在途、橙联可售等运营快照。

现有网页适合单个 SKU 补录，但不适合批量采购、分批打包、头程合并发货和海外仓部分上架。后续库存管理需要引入批次实体和批量流转工作台，同时保留完整历史。

## 2. 目标与非目标

### 2.1 第一期目标

1. 采购员可以一次创建采购批次，批量录入多个 SKU 和进货数量。
2. 已存在的 SKU 默认只引用 `01_SKU主数据`；只有主动点击“更新基础资料”后才能修改主档。
3. 新增货物自动进入 `本地仓待清点`。
4. 运营可以筛选、多选并批量推进尚未进入橙联可售状态的货物。
5. 同一个采购批次可以拆成多个物流批次；同一个物流批次也可以合并多个采购批次。
6. 同一条 SKU 明细允许只推进部分数量，系统自动拆分剩余数量。
7. 海外仓允许整批确认，也允许按 SKU 修正部分上架数量。
8. 清点、签收或上架数量不一致时，差异必须生成异常记录，不允许无痕消失。
9. 每次状态变化必须保留操作人、时间戳、前后状态和数量。
10. `19_SKU运营汇总` 只能由自动化更新，不允许人工维护。

### 2.2 第二期范围

- 飞书消息提醒和超时升级。
- Excel 批量导入模板。
- 扫码枪录入。
- 完整操作审计和处理时效看板。

### 2.3 非目标

- 第一期不实现采购审批、付款管理或供应商财务对账。
- 第一期不替代橙联仓库系统，只记录业务侧可见的签收和上架结果。
- 不回写或重算既有历史库存快照；新流程从启用日开始接管新增批次。

## 3. 已确认的业务规则

1. 采购批次允许拆分发货。
2. 海外仓偶尔会部分上架，因此保留 SKU 明细级修正能力。
3. 已存在 SKU 默认引用主档，显式进入编辑模式后才允许更新主档。
4. 国内处理中间阶段合并为 `待包装`。
5. 所有状态变化必须完整留痕。
6. 部分数量流转由系统自动拆分明细。
7. 数量差异自动生成异常。
8. 一个物流批次可以合并多个采购批次，采购批次与物流批次是多对多关系。

## 4. 数据模型

### 4.1 沿用表

#### `01_SKU主数据`

仅存储 SKU 基础档案，不写入在库、在途、销售或补货过程数据。

#### `02_库存流水`

继续作为不可覆盖的库存事件账本。扩展以下字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `流转事务号` | 文本 | 一次批量操作生成一个唯一事务号；用于幂等重试和成对流水关联 |
| `来源明细编号` | 文本 | 对应 `22_SKU批次库存明细` |
| `来源采购批次` | 文本 | 对应采购批次号 |
| `头程物流批次` | 文本 | 未绑定时为空 |
| `前状态` | 单选 | 状态更新前的明细状态 |
| `后状态` | 单选 | 状态更新后的明细状态 |
| `操作类型` | 单选 | 新增入库、状态推进、拆分、差异暂存、异常释放、报损、库存调整等 |
| `库存位置` | 单选 | 本地仓、国内集货仓、橙联在途、橙联可售、异常暂存 |
| `数量变动` | 数字 | 正数增加，负数减少 |
| `操作人` | 人员或文本 | 实际发起操作的用户 |
| `操作时间` | 日期时间 | 服务端生成 |
| `相关单号` | 文本 | 采购单、物流号、订单号等 |
| `备注` | 文本 | 补充说明 |

跨仓流转必须成对写入流水。例如从本地仓发出 `80` 件到国内集货仓：

| 流转事务号 | 库存位置 | 数量变动 |
| --- | --- | --- |
| `MOVE-20260602-0001` | 本地仓 | `-80` |
| `MOVE-20260602-0001` | 国内集货仓 | `+80` |

同一事务号下的流水在业务上必须完整。飞书写入中途失败时，接口必须返回“待重试”提示并保留事务号，由幂等重试补齐剩余步骤；不允许静默部分成功。

#### `19_SKU运营汇总`

保留已有字段，并增加：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `国内集货仓` | 数字 | 状态为已发往国内集货仓或国内集货仓待发的健康库存 |
| `异常暂存` | 数字 | 数量差异、报损待确认等暂存数量 |
| `账面总量` | 数字 | `总可用库存 + 异常暂存` |

`总可用库存` 的新口径为：

```text
本地库存 + 国内集货仓 + 橙联在途 + 橙联可售
```

### 4.2 新建表

#### `20_采购批次`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `采购批次号` | 文本，主字段 | 示例：`PO-202606-001` |
| `供应商` | 单选或文本 | 供应商名称 |
| `采购员` | 人员 | 录入责任人 |
| `下单日期` | 日期 | 采购日期 |
| `预计到货日期` | 日期 | 可选 |
| `实际到货日期` | 日期 | 可选 |
| `批次状态` | 单选 | 待录入、处理中、部分发出、已完成、异常 |
| `SKU种类数` | 公式 | 当前批次 SKU 数量 |
| `采购总件数` | 公式 | 当前批次原始件数 |
| `待处理件数` | 公式 | 尚未进入橙联可售或异常关闭的数量 |
| `备注` | 文本 | 可选 |

#### `21_头程物流批次`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `物流批次号` | 文本，主字段 | 示例：`SHIP-202606-001` |
| `承运商` | 文本 | 可选 |
| `跟踪号` | 文本 | 可选 |
| `发出日期` | 日期 | 发出头程日期 |
| `预计到仓日期` | 日期 | 可选 |
| `实际签收日期` | 日期 | 可选 |
| `物流状态` | 单选 | 待组建、国内集货仓待发、橙联在途、海外仓待上架、已完成、异常 |
| `来源采购批次` | 公式 | 从明细表汇总关联的采购批次号 |
| `SKU种类数` | 公式 | 当前物流批次 SKU 数量 |
| `发运总件数` | 公式 | 当前物流批次件数 |
| `备注` | 文本 | 可选 |

#### `22_SKU批次库存明细`

这是尚未完成流转货物的核心工作台。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `明细编号` | 文本，主字段 | 全局唯一，例如 `LOT-PO001-SKU001-01` |
| `来源采购批次` | 关联记录 | 关联 `20_采购批次` |
| `当前物流批次` | 关联记录 | 关联 `21_头程物流批次`，未组建物流时为空 |
| `SKU` | 文本 | 精确关联 SKU |
| `中文品名快照` | 文本 | 录入时保存，避免主档后续变化影响历史 |
| `原始数量` | 数字 | 初次创建时的数量 |
| `当前数量` | 数字 | 当前状态下仍需流转的数量 |
| `异常数量` | 数字 | 当前明细下尚未关闭的异常数量 |
| `当前状态` | 单选 | 使用第 5 节七段状态机 |
| `是否完成` | 复选框 | 进入橙联可售后勾选；默认工作台过滤掉已完成明细 |
| `最近操作人` | 人员或文本 | 最近一次更新责任人 |
| `最近更新时间` | 日期时间 | 服务端写入 |
| `版本号` | 数字 | 乐观锁，防止两个人同时推进同一明细 |
| `备注` | 文本 | 可选 |

`22` 是采购批次与物流批次之间的桥接表，因此天然支持多对多关联。

#### `23_库存异常`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `异常编号` | 文本，主字段 | 全局唯一 |
| `来源明细编号` | 文本 | 对应 `22` |
| `SKU` | 文本 | 对应商品 |
| `异常类型` | 单选 | 清点差异、集货仓签收差异、海外仓签收差异、上架差异、报损、其他 |
| `责任节点` | 单选 | 产生差异的流程节点 |
| `预期数量` | 数字 | 操作前应有数量 |
| `实收数量` | 数字 | 实际确认数量 |
| `差异数量` | 数字 | `实收数量 - 预期数量` |
| `处理状态` | 单选 | 待处理、处理中、已补回、已报损、已关闭 |
| `负责人` | 人员 | 异常跟进人 |
| `创建时间` | 日期时间 | 自动写入 |
| `关闭时间` | 日期时间 | 可选 |
| `备注` | 文本 | 可选 |

## 5. 七段状态机

| 顺序 | 明细状态 | 汇总位置 | 说明 |
| --- | --- | --- | --- |
| 1 | `本地仓待清点` | 本地库存 | 采购员录入后自动进入 |
| 2 | `待包装` | 本地库存 | 清点、质检、贴标和包装统一处理 |
| 3 | `已发往国内集货仓` | 国内集货仓 | 公司发出后进入 |
| 4 | `国内集货仓待发` | 国内集货仓 | 集货仓签收后进入 |
| 5 | `橙联在途` | 橙联在途 | 绑定头程物流批次并确认发运后进入 |
| 6 | `海外仓待上架` | 橙联在途 | 海外仓签收但尚未上架 |
| 7 | `橙联可售` | 橙联可售 | 上架确认后并入可售库存 |

只允许按顺序推进。库存调整、回退、报损和异常释放使用独立操作类型，并要求填写原因，不通过普通“推进状态”按钮完成。

## 6. 自动化流转规则

### 6.1 采购批次录入

1. 采购员新建采购批次。
2. 批量添加 SKU 与数量。
3. 已存在 SKU 默认引用主档；新 SKU 必须补齐主档后才能加入批次。
4. 保存后为每个 SKU 创建一条 `22_SKU批次库存明细`，状态为 `本地仓待清点`。
5. 同时向 `02_库存流水` 写入本地仓正数入库流水。
6. 刷新 `19_SKU运营汇总` 的本地库存。

### 6.2 普通状态推进

1. 用户在网页工作台按状态、采购批次或物流批次筛选明细。
2. 多选明细，选择目标状态。
3. 服务端校验目标状态是否为合法下一步。
4. 整行推进时直接更新明细；部分推进时创建一条新明细承接推进数量，原明细保留剩余数量。
5. 如果汇总位置发生变化，向 `02_库存流水` 写入一减一增的成对流水。
6. 刷新相关 SKU 的 `19_SKU运营汇总`。

### 6.3 组建头程物流批次

1. 用户创建 `21_头程物流批次`。
2. 从 `国内集货仓待发` 明细中多选货物。
3. 明细可以来自多个采购批次。
4. 支持整行绑定或部分数量绑定；部分绑定时自动拆分明细。
5. 确认发运后批量推进为 `橙联在途`。

### 6.4 海外仓签收与上架

1. 海外仓签收时默认整批确认。
2. 如有差异，可以按 SKU 修改实收数量。
3. 签收数量进入 `海外仓待上架`。
4. 上架时默认整批确认，也允许按 SKU 输入部分上架数量。
5. 已上架数量推进到 `橙联可售`，并并入可售库存。

### 6.5 数量差异

1. 预期数量与实收数量不一致时，系统自动创建 `23_库存异常`。
2. 差异数量通过 `02_库存流水` 转入 `异常暂存`，不得直接删除。
3. 异常关闭时，必须明确选择“补回库存”或“确认报损”。
4. 补回库存时写入异常暂存减少和目标位置增加的成对流水。
5. 确认报损时写入异常暂存减少流水并保留报损记录。

## 7. 网页端工作台

新增导航模块：`库存流转`。

### 7.1 `采购批次`

- 新建采购批次。
- 批量录入 SKU 与数量。
- 搜索并引用已有 SKU 主档。
- 新 SKU 补档。
- 显式进入主档编辑模式。

### 7.2 `批次流转`

- 按状态、采购批次、物流批次、SKU 筛选。
- 多选明细批量推进。
- 单个明细支持输入部分流转数量。
- 批量操作前显示影响范围：SKU 数、明细数、总件数、目标状态。

### 7.3 `头程物流`

- 新建物流批次。
- 从国内集货仓待发明细中跨采购批次多选货物。
- 填写承运商、跟踪号、发出日期和预计到仓日期。
- 整批确认头程发运、海外仓签收和上架。
- 出现差异时展开 SKU 明细修正。

### 7.4 `库存异常`

- 查看待处理异常。
- 按异常类型、责任节点、负责人和创建时间筛选。
- 分配负责人。
- 选择补回库存或确认报损后关闭异常。

### 7.5 保留现有 `数据录入`

现有页面继续用于：

- SKU 主数据录入。
- 销售日报。
- 零散库存调整。
- 客服异常。
- 竞品监控。

## 8. 服务端编排与一致性

核心库存扣增不完全交给飞书工作流。网页服务端负责：

1. 生成全局唯一 `流转事务号`。
2. 校验状态机、数量、必填物流批次和明细版本号。
3. 执行明细拆分或更新。
4. 成对追加库存流水。
5. 刷新运营汇总。
6. 创建异常记录。

由于飞书多维表格不提供跨表数据库事务，服务端需要使用幂等策略：

- 每次批量操作使用唯一事务号。
- 写入前按事务号检查是否已处理。
- 同一明细使用 `版本号` 做乐观锁。
- 批量写入分批串行执行，单批不超过 `200` 条。
- 如果中途失败，接口返回“待重试”状态并保留事务号；重试时跳过已完成步骤。
- 不允许在同步失败时无提示地返回成功。

## 9. 飞书公式与工作流分工

### 9.1 公式字段

飞书公式用于长期展示：

- 采购批次 SKU 种类数、采购总件数、待处理件数。
- 物流批次来源采购批次、SKU 种类数、发运总件数。
- 超时天数和超时标识。
- 运营汇总的账面总量。

### 9.2 飞书工作流

第二期使用飞书工作流发送提醒：

- `本地仓待清点` 超过 `2` 天。
- `待包装` 超过 `3` 天。
- `国内集货仓待发` 超过 `5` 天。
- 已超过预计到仓日期但仍未签收。
- 异常记录超过 `3` 天未关闭时升级提醒管理员。

核心库存扣增、部分数量拆分和汇总刷新仍由网页服务端处理。

## 10. 权限模型

- 采购员：可创建采购批次、补充新 SKU 主档、查看本地阶段明细。
- 运营：可推进国内阶段、创建物流批次、查看异常。
- 管理员：可处理回退、库存调整、异常关闭和报损。
- `19_SKU运营汇总`：普通用户只读，网页服务端应用身份可写。
- `02_库存流水`：普通用户只读，网页服务端应用身份追加写入。

网页服务端需要对每次操作保存当前登录用户，不能只记录应用机器人。

## 11. 迁移与上线策略

1. 新建 `20` 至 `23` 四张表。
2. 扩展 `02_库存流水` 和 `19_SKU运营汇总` 字段。
3. 不迁移既有历史批次，不修改当前库存快照。
4. 设定启用时间点；启用时间点之后的新增采购批次走新流程。
5. 上线前用测试 SKU 创建一组采购批次，覆盖整批推进、部分拆分、物流合并、海外仓差异和异常关闭。
6. 测试通过后再让真实新增采购切换到新流程。

## 12. 异常处理

- 非法跳级：拒绝操作并提示允许的下一状态。
- 数量超限：拒绝操作并显示当前可推进数量。
- 并发修改：版本号不一致时拒绝保存，要求刷新页面。
- 缺少物流批次：进入橙联在途前拒绝操作。
- 汇总同步失败：返回待重试事务号，不允许静默成功。
- 差异记录缺少处理结果：不允许关闭异常。
- 飞书权限不足：返回明确表名与错误码，避免只显示通用失败。

## 13. 验收测试

### 13.1 正常链路

1. 新建一个采购批次，录入多个已有 SKU 和一个新 SKU。
2. 验证货物自动进入本地仓待清点，本地库存增加。
3. 批量推进到待包装。
4. 批量推进到国内集货仓并验证本地仓减少、国内集货仓增加。
5. 合并两个采购批次建立一个物流批次。
6. 确认发运并验证国内集货仓减少、橙联在途增加。
7. 确认海外仓签收和上架并验证橙联在途减少、橙联可售增加。

### 13.2 拆分与异常

1. 对 `100` 件明细仅推进 `80` 件，验证原明细保留 `20` 件，新明细推进 `80` 件。
2. 海外仓预期签收 `100` 件，实收 `97` 件，验证生成 `-3` 件异常暂存。
