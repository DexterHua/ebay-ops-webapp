# SKU 详情查询页设计

日期：2026-06-07

## 1. 背景

运营需要在系统内快速输入 SKU、中文品名、英文标题关键词、OEM 等任意片段，联想并选中匹配 SKU，集中查看商品资料、价格、图片和库存状态。当前库存监控页已经会合并读取 `01_SKU主数据`、`18_SKU库存策略`、`19_SKU运营汇总`，本功能复用这条只读数据路径。

## 2. 目标

1. 新增菜单项 `SKU 详情`，路径为 `/sku-details`。
2. 支持按 SKU 任意字段模糊搜索并显示联想结果。
3. 选中 SKU 后展示商品名称、英文标题关键词、OEM、商品尺寸、重量、商品图片链接、建议售价、目前库存数量、在途数量、预计可售天数等信息。
4. 页面只读，不写入飞书多维表格。
5. 保持现有后台页面风格、权限和移动端菜单行为。

## 3. 非目标

1. 不在一期新增飞书写入、字段编辑或图片上传能力。
2. 不新建专用搜索 API；当前 SKU 规模下先由前端读取并合并三张表。
3. 不改变库存监控、库存流转或销售扫描的现有数据口径。

## 4. 信息架构

### 4.1 菜单

在 `MODULES` 中新增：

| 字段 | 值 |
| --- | --- |
| `id` | `skuDetails` |
| `name` | `SKU 详情` |
| `path` | `/sku-details` |
| `description` | `按 SKU、品名、OEM 等字段查询商品与库存详情` |

该模块不设置 `adminOnly`，采购与运营角色都可查看。

### 4.2 页面结构

页面包含四个区域：

1. 标题与刷新按钮。
2. 搜索输入框与联想结果列表。
3. 选中 SKU 的详情面板。
4. 加载、错误、无匹配、未选中等空状态。

## 5. 数据流

页面加载时并行读取：

- `/api/lark?table=sku`
- `/api/lark?table=strategy`
- `/api/lark?table=summary`

前端以 `SKU` 为键合并记录：

- 主数据提供商品名称、英文标题关键词、OEM、尺寸、重量、图片、建议售价、类目、状态、供应商等字段。
- 库存策略提供安全库存、补货周期天数等字段。
- 运营汇总提供橙联可售、橙联在途、本地库存、国内集货仓、总可用库存、近 7 日日均销量、可售天数或预计可售天数等字段。

图片字段兼容以下名称，取第一个可用 URL：

- `商品图片（链接）`
- `商品图片`
- `图片链接`
- `Image URL`
- `imageUrl`

## 6. 搜索规则

搜索输入去掉前后空格并转为小写后，对每个 SKU 的以下字段做包含匹配：

- `SKU`
- `中文品名`
- `英文标题关键词`
- `OEM`
- `类目`
- `供应商`
- `SKU状态`

默认显示前 8 个匹配结果。没有输入时显示最近加载的前 8 个 SKU，方便运营快速点选。

## 7. 展示字段

详情面板展示：

| 分组 | 字段 |
| --- | --- |
| 商品资料 | SKU、中文品名、英文标题关键词、OEM、类目、SKU 状态、供应商 |
| 规格价格 | 建议售价、采购价、预估毛利率、商品毛重（g）、商品尺寸（含包装）（cm） |
| 库存销售 | 橙联可售、橙联在途、本地库存、国内集货仓、总可用库存、近 7 日日均销量、预计可售天数、安全库存、补货周期天数、补货状态 |
| 图片 | 图片预览、图片链接、复制链接入口 |

数字为空时显示 `--`，库存数量按中国区域格式化；价格使用 `$` 展示建议售价，采购价使用 `¥`。

## 8. 错误处理

1. 任一飞书请求失败时展示错误卡片和刷新按钮。
2. 三张表读取成功但没有 SKU 时展示空状态。
3. 搜索没有匹配时展示无结果提示。
4. 图片 URL 为空时展示无图片占位；图片加载失败时保留链接文本供人工打开。

## 9. 测试

1. 单元测试覆盖数字解析、字段文本化、图片 URL 提取、三表合并和模糊搜索。
2. 页面代码通过 TypeScript 与 lint 检查。
3. 构建验证使用现有 `npm run build`；如时间允许再运行 Cloudflare 构建。
